    介绍(Overview)

    欢迎使用BTC.com浏览器API！

    欢迎使用BTC.com提供的区块链数据API，使开发者能够通过RestFul实时获取比特币链上区块、交易、地址等数据。

    BTC.com的Bitcoin RPC节点，每月处理数千亿的请求。该文档方便开发人员与BTC.com集成，展示了如何使用cURL、Python和任何适用的Web3 SDK调用RPC的方法。

    Restful API请求结构

    获取地址、交易和区块相关信息，通过RestFul方式调用，调用路径如下：

    https://$(Endpoint)/${method}/${version}/${model}/${path}

    其中：

    键 值 说明
    Endpoint tools-gateway.api.btc.com
    method rest/api
    version v1.0
    model nodeapi
    path API路径，参见具体请求路径

    节点RPC请求结构

    获取节点信息，采用RPC调用方式，调用路径如下：

    https://$(Endpoint)/${method}/${version}/${model}

    其中：

    键 值 说明
    Endpoint tools-gateway.api.btc.com
    method rpc/api
    version v1.0
    model noderpc

    鉴权

    调用Restful API和RPC请求时，需要在Header中传递X-API-TOKEN(Access Key)进行鉴权。

    X-API-TOKEN(Access Key) 是用户身份凭据，对应一个账户，请用户保管好自己的API Token(Access Key);

    X-API-TOKEN(Access Key)可以在tools.btc.com登录账户后，在API管理页面获取。

    响应

    所有响应类型均为 application/json

    Restful API响应格式如下：

    键 值 说明
    code 0 API响应状态码
    msg success code对应的描述信息
    data {} 具体API响应的数据
    timestamp 1699966857 请求花费的时间

    RPC响应格式如下：

    键 值 说明
    jsonrpc 2.0 jsonrpc版本
    result "" 请求返回值
    error "" 错误描述
    id "1699966857" 请求ID，为空时，以时间戳代替
